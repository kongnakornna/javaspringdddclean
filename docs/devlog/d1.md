# 📊 วิเคราะห์ปัญหา Circular Dependency ใน Spring Boot และแนวทางแก้ไข

จาก Log Error ที่แสดง:

```
The dependencies of some of the beans in the application context form a cycle:

   jwtTokenFilter defined in file [...]
      ↓
   customUserDetailsService defined in file [...]
┌─────┐
|  userRepositoryImpl defined in file [...]
└─────┘
```

---

## 🔍 สรุปปัญหา (Root Cause Analysis)

### ภาษาไทย
ระบบพบ **Circular Dependency (การอ้างอิงแบบวนรอบ)** ระหว่าง Bean 3 ตัว:
1. `jwtTokenFilter` → ต้องการ `customUserDetailsService`
2. `customUserDetailsService` → ต้องการ `userRepositoryImpl`
3. `userRepositoryImpl` → ต้องการ `jwtTokenFilter` (โดยทางอ้อม)

Spring Boot 2.6+ **ห้าม Circular Dependency โดยค่าเริ่มต้น** (`allow-circular-references=false`) ทำให้ Application Start ไม่ได้

### ภาษาอังกฤษ
The system encountered a **Circular Dependency** between 3 beans:
1. `jwtTokenFilter` → depends on `customUserDetailsService`
2. `customUserDetailsService` → depends on `userRepositoryImpl`
3. `userRepositoryImpl` → indirectly depends on `jwtTokenFilter`

Spring Boot 2.6+ **prohibits circular references by default** (`allow-circular-references=false`), causing the application to fail startup.

---

## 🛠️ แนวทางแก้ไข (Solutions)

### วิธีที่ 1: เปิด Circular Reference (แก้ไขเร็ว - ไม่แนะนำสำหรับ Production)

**ไฟล์: `application.yml` หรือ `application.properties`**
```yaml
# application.yml
spring:
  main:
    allow-circular-references: true
```

```properties
# application.properties
spring.main.allow-circular-references=true
```

⚠️ **ข้อเสีย**: เป็นการปิดบังปัญหา ไม่ได้แก้ที่ต้นเหตุ อาจทำให้เกิดปัญหาอื่นๆ ในอนาคต

---

### วิธีที่ 2: ใช้ @Lazy (แนะนำสำหรับกรณีที่ต้องพึ่งพากัน)

ให้ Inject Dependency แบบ Lazy (สร้างตอนใช้งานจริง)

#### แก้ไข `JwtTokenFilter.java`
```java
@Component
@Profile({"dev", "prod"})
public class JwtTokenFilter extends OncePerRequestFilter {

    private final LogService logService;
    private final CustomUserDetailsService customUserDetailsService; // ✅ เปลี่ยนเป็น Lazy

    @Value("${jwt.privateKey}")
    private String privateKey;

    @Value("${jwt.publicKey}")
    private String publicKey;

    // ✅ ใช้ Constructor Injection + @Lazy
    public JwtTokenFilter(LogService logService, 
                          @Lazy CustomUserDetailsService customUserDetailsService) {
        this.logService = logService;
        this.customUserDetailsService = customUserDetailsService;
    }
    // ...
}
```

#### หรือใช้ Setter Injection + @Lazy
```java
@Component
@Profile({"dev", "prod"})
public class JwtTokenFilter extends OncePerRequestFilter {

    private LogService logService;
    private CustomUserDetailsService customUserDetailsService;

    @Autowired
    public JwtTokenFilter(LogService logService) {
        this.logService = logService;
    }

    @Autowired
    @Lazy
    public void setCustomUserDetailsService(CustomUserDetailsService customUserDetailsService) {
        this.customUserDetailsService = customUserDetailsService;
    }
    // ...
}
```

### วิธีที่ 3: ใช้ Setter Injection แทน Constructor Injection (ในคลาสที่สร้าง Cycle)

#### แก้ไข `CustomUserDetailsService.java`
```java
@Service
public class CustomUserDetailsService implements UserDetailsService {

    private UserRepository userRepository; // ✅ เปลี่ยนเป็น Field Injection หรือ Setter

    // ✅ ใช้ Setter Injection
    @Autowired
    public void setUserRepository(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        // ใช้ userRepository
    }
}
```

### วิธีที่ 4: แยก Logic ออกจากกัน (Refactoring - แนะนำที่สุด)

#### 4.1 สร้าง Interface กลาง
```java
// สร้าง Interface ใหม่
public interface UserService {
    UserDetails loadUserByUsername(String username);
}

// CustomUserDetailsService Implement Interface
@Service
public class CustomUserDetailsService implements UserService {
    // ...
}
```

#### 4.2 JwtTokenFilter ใช้ Interface แทน Implementation
```java
@Component
public class JwtTokenFilter extends OncePerRequestFilter {
    
    private final UserService userService; // ✅ ใช้ Interface แทน

    public JwtTokenFilter(UserService userService) {
        this.userService = userService;
    }
    // ...
}
```

#### 4.3 แยก `UserRepository` และ `UserService` ออกจากกัน
```java
// UserService ไม่ต้องพึ่ง JwtTokenFilter โดยตรง
@Service
public class UserService {
    private final UserRepository userRepository;
    
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

// JwtTokenFilter ใช้ UserService
@Component
public class JwtTokenFilter {
    private final UserService userService; // ✅ ไม่มี Cycle แล้ว
}
```

### วิธีที่ 5: ใช้ ApplicationContext.getBean() (เฉพาะกรณีฉุกเฉิน)

```java
@Component
public class JwtTokenFilter extends OncePerRequestFilter {

    @Autowired
    private ApplicationContext applicationContext;

    private CustomUserDetailsService customUserDetailsService;

    // ✅ ดึง Bean ตอนใช้งาน (Runtime)
    private CustomUserDetailsService getCustomUserDetailsService() {
        if (customUserDetailsService == null) {
            customUserDetailsService = applicationContext.getBean(CustomUserDetailsService.class);
        }
        return customUserDetailsService;
    }

    @Override
    protected void doFilterInternal(...) {
        CustomUserDetailsService userDetailsService = getCustomUserDetailsService();
        // ใช้ userDetailsService
    }
}
```

| ลำดับ | วิธี | ระดับความยาก | ความปลอดภัย | เหมาะกับ |
|-------|-----|-------------|-------------|----------|
| 1 | `@Lazy` | ต่ำ | ⭐⭐⭐⭐ | แก้ไขเร็ว ปลอดภัย |
| 2 | Setter Injection | ต่ำ | ⭐⭐⭐⭐ | แก้ไขง่าย |
| 3 | Refactoring แยก Interface | ปานกลาง | ⭐⭐⭐⭐⭐ | **แนะนำที่สุด** |
| 4 | `allow-circular-references=true` | ต่ำ | ⭐⭐ | ไม่แนะนำ (ปิดบังปัญหา) |

## ✅ วิธีที่แนะนำที่สุด: ใช้ @Lazy + Refactoring

### ขั้นตอนที่ 1: เพิ่ม `@Lazy` ใน `JwtTokenFilter`

```java
// com/icmon/module/auth/infrastructure/security/JwtTokenFilter.java
@Component
@Profile({"dev", "prod"})
public class JwtTokenFilter extends OncePerRequestFilter {

    private final LogService logService;
    private final CustomUserDetailsService customUserDetailsService; // ✅ ยังคงเป็น final

    @Value("${jwt.privateKey}")
    private String privateKey;

    @Value("${jwt.publicKey}")
    private String publicKey;

    // ✅ ใช้ @Lazy ใน Constructor
    public JwtTokenFilter(LogService logService, 
                          @Lazy CustomUserDetailsService customUserDetailsService) {
        this.logService = logService;
        this.customUserDetailsService = customUserDetailsService;
    }
    // ...
}
```

### ขั้นตอนที่ 2: ตรวจสอบ `UserRepositoryImpl` ว่าไม่ได้พึ่ง `JwtTokenFilter`

ดูที่ `UserRepositoryImpl` ว่า Inject อะไรเข้าไป:
```java
// com/icmon/module/auth/infrastructure/repository/impl/UserRepositoryImpl.java
@Repository
public class UserRepositoryImpl implements UserRepository {
    
    // ✅ ตรวจสอบว่าไม่มี @Autowired JwtTokenFilter
    // ❌ ถ้ามี ให้ลบออก (เพราะ Repository ไม่ควรพึ่ง Security Filter)
}
```
